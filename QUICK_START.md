# 🚀 התחלה מהירה - איך להריץ את הסקריפט?

## שלב 1: התקנה (פעם אחת)

פתח PowerShell או CMD וכתוב:

```bash
cd c:\Users\Asus-pc1\OneDrive\Desktop\devops-project
npm install
```

זה יתקין את כל הספריות הנדרשות.

---

## שלב 2: הרצת הסקריפט

### אופציה 1: ניתוח קובץ לוג יחיד
```bash
npm start -- --file logs/sample.log
```

**מה זה עושה?**
- קורא את `logs/sample.log`
- בודק שהקובץ תקין (ולידציות)
- מנתח את כל השורות (חישובים)
- מדפיס דוח עם התוצאות

### אופציה 2: שמירת דוח ל-JSON
```bash
npm start -- --file logs/sample.log --format json --output result.json
```

זה ייצור קובץ `result.json` עם כל התוצאות.

### אופציה 3: יצירת דוח HTML
```bash
npm start -- --file logs/sample.log --format html --output report.html
```

זה ייצור קובץ `report.html` שתוכל לפתוח בדפדפן.

## מה אם יש שגיאה?

### שגיאה: "קובץ לא נמצא"
```
❌ קובץ לא נמצא: logs/sample.log
```
**פתרון:** ודא שהקובץ קיים בתיקייה `logs/`

### שגיאה: "אין הרשאות"
```
❌ אין הרשאות קריאה לקובץ
```
**פתרון:** ודא שיש לך הרשאות לקרוא את הקובץ

### שגיאה: "קובץ גדול מדי"
```
❌ קובץ גדול מדי: 150MB (מקסימום: 100MB)
```
**פתרון:** ערוך את `config/config.json` והגדל את `maxFileSize`

## איך זה עובד עם Jenkins?

כשאתה דוחף קוד ל-GitHub, Jenkins:

1. ✅ שולף את הקוד
2. ✅ מתקין תלויות (`npm install`)
3. ✅ מריץ בדיקות (`npm test`)
4. ✅ יוצר קבצי לוג לדוגמה
5. ✅ מריץ את הסקריפט (`npm start`)
6. ✅ שומר את התוצאות כ-artifacts

**כל זה קורה אוטומטית!**
